✅ AI/ML FEATURES
═══════════════════════════════════════════════════════════════════════════════

🚨 FRAUD DETECTION MODEL
   Input: 6 risk features
   Output: Fraud probability (0-100%), Decision, Confidence, SHAP Explanation
   Model: XGBoost-style Gradient Boosting
   Features: Amount, Velocity, Location Risk, Device Risk, Time Anomaly, Network Risk
   Decisions: APPROVED (< 50%) | STEP-UP AUTH (50-75%) | BLOCKED (> 75%)

📊 TRUST SCORING ENGINE
   Input: 5 risk components
   Output: Trust score (0-100), Decision
   Formula: 0.2·Identity + 0.2·Auth + 0.25·Transaction + 0.2·Network + 0.15·Behavioral
   Decisions: APPROVE (> 75%) | STEP-UP AUTH (50-75%) | BLOCK (< 50%)

🔍 AML PATTERN DETECTION
   5 Money Laundering Patterns:
   • Layering    - Multiple transactions in short time with varying amounts
   • Structuring - Amounts just below thresholds (repeated pattern)
   • Smurfing    - Many small transactions (velocity-based)
   • Dispersion  - Transactions to many different recipients
   • Circular    - Circular transaction patterns
   Output: Individual scores + Overall Risk Level (LOW/MEDIUM/HIGH/CRITICAL)
   Auto-generates STR (Suspicious Transaction Report) when high risk detected

🆔 KYC VERIFICATION
   Modules: OCR Extraction, Tamper Detection, Synthetic Identity Check, Consent Validation
   Score: Overall risk percentage
   Result: APPROVED / PENDING / REJECTED

🔐 AUTHENTICATION (Multi-Layer)
   Verification Methods:
   • Face Recognition + Deepfake Detection
   • Liveness Detection
   • Voice Biometrics
   Decision: ALLOW / BLOCK based on all layers passing

👤 BEHAVIORAL BIOMETRICS
   Metrics: Typing Velocity (WPM), Mouse Movement (px/s), Keystroke Patterns
   Anomaly Detection: Flags unusual behavior patterns

💡 EXPLAINABLE AI (SHAP)
   Provides feature importance explanations for all ML predictions
   Shows which factors influenced each decision
